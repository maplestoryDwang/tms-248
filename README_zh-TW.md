## 說明
### 感恩
感謝所有願意分享和熱愛楓之谷的朋友，本倉庫原始地址：https://github.com/rage123450/TMS246-Server

### 專案說明
1. TMS-248.1版本伺服器端，兩年前在網路上找到的
2. 基於swordie架構的TMS伺服器端
3. 使用jdk17
4. 資料庫：mariadb
5. 啟動方式可以參考其他TMS伺服器的啟動方式

### 為什麼要做這個專案
1. 雖然TMS245的魔森端已經非常優秀，但我個人更偏好swordie的專案架構。重構專案是一個大工程，正好看到有這麼一個專案，嘗試運行後發現能夠正常啟動。
2. 之前在研究GMS214，雖然也很優秀，但遊戲解析度讓我頭痛，4K螢幕看起來非常不舒服，又不想透過HOOK修改記憶體，所以想選擇一個較高版本進行修復。
3. 目前已經有很多優秀的伺服器端可供參考。僅憑靜態調試已完全滿足修復需求。

### 額外說明
1. 請注意dat資料的儲存路徑和檔案名稱，否則容易閃退
2. 專案缺少許多腳本，需要進行修復。
3. 目前仍以GMS214為主。修復完成的功能如果適用於此專案，就會merge過來。

### 快速開始
1. 確保已安裝JDK 17或更高版本
2. 安裝並配置MariaDB資料庫
3. 匯入專案所需的資料庫檔案
4. 配置伺服器設定
5. 啟動伺服器

### 系統需求
- JDK 17+
- MariaDB 10.3+
- 至少4GB記憶體
- Windows作業系統
